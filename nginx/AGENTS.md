# Nginx 配置目录 (nginx/)

## 目录概述
nginx/ 目录用于存储 Nginx 反向代理配置文件，为生产环境提供 Web 服务器和负载均衡功能。

## 文件结构
```
nginx/
├── AGENTS.md              # 本文档 - AI 代理工作指令
└── nginx.conf            # 主要 Nginx 配置文件
```

## 主要组件

### nginx.conf
- **用途**: 生产环境 Nginx 配置
- **服务**: docker-compose.prod.yml
- **功能**:
  - API 代理转发 (api:8080)
  - 安全头设置
  - 文件上传代理
  - 健康检查端点

## AI 代理工作指令

### DevOps 工程师代理 (DevOps Engineer)
- **职责**: 负责 Nginx 配置的部署和维护
- **工作内容**:
  - 监控 Nginx 服务状态
  - 优化配置性能
  - 处理 SSL/TLS 证书管理

### 网络架构代理 (Network Architect)
- **职责**: 负责负载均衡和代理配置
- **工作内容**:
  - 配置上游服务器集群
  - 优化代理超时设置
  - 设置健康检查机制

### 安全专家代理 (Security Expert)
- **职责**: 负责 Web 服务器安全配置
- **工作内容**:
  - 配置安全响应头
  - 实施 IP 黑名单策略
  - 设置访问控制规则

### 性能优化代理 (Performance Optimizer)
- **职责**: 负责 Nginx 性能调优
- **工作内容**:
  - 优化连接数和缓存
  - 配置 gzip 压缩
  - 监控响应时间

## 配置特性
- **API 代理**: 将 /api/ 请求转发到后端服务
- **文件上传**: 处理 /uploads/ 文件上传请求
- **安全防护**: 配置标准安全响应头
- **健康检查**: 提供 /health 端点服务监控

## 工作流程
1. 配置编写 → 安全审查 → 性能测试 → 生产部署 → 监控维护

## 注意事项
- 配置变更需要在测试环境验证
- 定期检查日志文件和错误率
- 保持配置与 docker-compose.prod.yml 同步
